## Stack context (for executors)

VibeOps is a Docker LEMP ops stack with a pure-Python stdlib CLI (`./manage.py`).

- **Verification gate**: `make check` (syntax + `python3 -B -m unittest discover -s tests -v`)
- **Layering**: `utils` → `os` → `services` → `ui` → `commands` (see `docs/architecture.md`)
- **State**: `runtime/state/stack.json` is source of truth; `runtime/generated/` is disposable
- **Render**: mutations that change generated config call `apply_generated_config` with selective `service_targets`
- **Conventions**: no third-party runtime deps; no `from x import *`; commit messages are short imperative phrases (e.g. `access log`, `enhance cron`)
